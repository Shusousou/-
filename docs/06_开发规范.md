## 1. 命名规范

### 1.1 Python代码
| 类型 | 规范 | 示例 |
|------|------|------|
| 变量名 | snake_case（小写+下划线） | user_name, book_list |
| 函数名 | snake_case | get_user_by_id() |
| 类名 | PascalCase（首字母大写） | UserModel, BookSearch |
| 文件名 | snake_case | library_api.py |
| 常量 | UPPER_CASE | MAX_RETRY_COUNT = 3 |

### 1.2 数据库
| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写复数 | users, exchange_books |
| 字段名 | snake_case | created_at, is_verified |

### 1.3 URL路径
| 类型 | 规范 | 示例 |
|------|------|------|
| 路径 | 小写+斜线 | /search/result |
| 参数 | 小写+下划线 | ?isbn=xxx&page=1 |

## 2. 代码组织规范

### 2.1 每个模块的结构
`
模块名/
├── __init__.py        # 空文件（标识这是一个Python包）
├── routes.py          # 路由定义（URL → 函数）
└── templates/         # 本模块的HTML模板
`

### 2.2 routes.py 文件模板
`python
from fastapi import APIRouter, Request
from fastapi.responses import HTMLResponse
from fastapi.templating import Jinja2Templates

router = APIRouter(prefix="/模块路径", tags=["模块名"])
templates = Jinja2Templates(directory="shusousou/modules/模块名/templates")

@router.get("/", response_class=HTMLResponse)
async def 页面名称(request: Request):
    \"\"\"函数说明\"\"\"
    # 业务逻辑
    return templates.TemplateResponse("模板名.html", {"request": request, ...})
`

### 2.3 HTML模板规范
- 使用 base.html 作为公共模板（包含导航栏、页脚）
- 每个页面继承 base.html
- 模板变量使用双花括号 {{ variable }}
- 中英文文案使用字典变量传递

## 3. 文件编码
- 所有文件使用 **UTF-8** 编码
- Python文件头部不需要写编码声明（Python 3默认UTF-8）

## 4. 注释规范
- 函数需要写文档字符串（"""说明"""）
- 复杂逻辑需要写行内注释
- 注释用中文

`python
def search_books(keyword: str) -> list:
    \"\"\"根据关键词搜索图书馆图书
    
    Args:
        keyword: 搜索关键词
    
    Returns:
        图书列表，每本包含标题、作者、状态等信息
    \"\"\"
    # 这里是具体的搜索逻辑
    ...
`

## 5. 错误处理规范
- 用户输入错误 → 返回友好提示（不要崩页面）
- API调用失败 → 显示"服务暂时不可用，请稍后再试"
- 数据库错误 → 记录日志，显示"系统错误，请联系管理员"
- 使用 try/except 捕获异常

## 6. Git规范（后续使用）
- main 分支：稳定版本
- dev 分支：开发中
- 功能开发在 feature/xxx 分支
- 提交信息格式：[模块] 做了什么
