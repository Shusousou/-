## 1. 测试原则
- **每完成一个功能，立即测试**
- **先手动测试，后续再考虑自动化**
- **测试环境 = 开发环境**（本地运行即可测试）

## 2. 测试流程

### 2.1 每步开发完成后
1. 运行程序：python shusousou/main.py
2. 打开浏览器：http://localhost:8000
3. 按测试用例逐项测试
4. 发现问题 → 记日志 → 修复
5. 全部通过 → 进入下一步

### 2.2 测试模板
```
## 测试：{功能名称}

### 前置条件
- 已登录用户：{用户名}
- 数据库中已有数据：{说明}

### 测试步骤
1. 打开页面：{URL}
2. 操作：{具体操作}
3. 预期结果：{期望看到什么}
4. 实际结果：{实际看到什么}
5. 结论：通过 / 不通过（原因）
```

## 3. 各阶段测试重点

### Phase 0: 项目初始化
| 测试项 | 预期结果 |
|--------|---------|
| 运行 main.py | 无报错 |
| 打开 http://localhost:8000 | 看到首页 |
| 首页导航栏 | 显示所有菜单项 |

### Phase 1: 用户系统
| 测试项 | 预期结果 |
|--------|---------|
| 注册（正确信息） | 提示注册成功，收到验证邮件 |
| 注册（邮箱已存在） | 提示邮箱已被注册 |
| 注册（密码太短） | 提示密码太短 |
| 验证邮箱 | 提示验证成功，可登录 |
| 登录（正确信息） | 登录成功，跳转首页 |
| 登录（错误密码） | 提示密码错误 |
| 未验证邮箱登录 | 提示请先验证邮箱 |
| 退出 | 退出成功，显示未登录状态 |

### Phase 2: 书搜搜
| 测试项 | 预期结果 |
|--------|---------|
| 输入空prompt | 提示请输入内容 |
| 输入"机器学习" | 显示推荐卡片（在馆状态） |
| 输入"稀有冷门书" | 显示推荐卡片（无匹配状态） |
| 点击语言切换 | 界面切换为英文 |
| 搜索结果包含论坛评论 | 卡片下方显示评论 |
| 搜索结果包含交换信息 | 卡片下方显示交换按钮 |

### Phase 3: 论坛
| 测试项 | 预期结果 |
|--------|---------|
| 发帖 | 帖子显示在列表中 |
| 发帖（空内容） | 提示不能为空 |
| 回帖 | 评论显示在帖子下方 |
| 点赞 | 点赞数+1，不能重复点赞 |
| 搜索帖子 | 显示匹配结果 |
| 分类筛选 | 只显示该类帖子 |
| 帖子显示图书馆状态 | 显示该书的在馆状态 |

### Phase 4: 交换
| 测试项 | 预期结果 |
|--------|---------|
| 发布图书 | 显示在可借列表中 |
| 浏览列表 | 显示所有可借图书 |
| 搜索 | 显示匹配的图书 |
| 点击想借 | 进入临时评论区 |
| 发送消息 | 对方能看到消息 |
| 挂Request | Request显示在列表中 |
| 发布匹配的书 | 触发邮件通知 |

### Phase 5: 动态模拟
| 测试项 | 预期结果 |
|--------|---------|
| 多次搜索同一本书 | 状态可能变化 |
| 查看不同时间的数据 | 借阅数据有变化 |
