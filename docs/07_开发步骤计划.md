## 开发原则
1. **小步快跑**：每次只做一个功能，做完再推进下一个
2. **随时可用**：每一步做完，网站都是可以运行的
3. **文档同步**：代码更新了，文档也要更新
4. **日志记录**：每天记录进度和问题

### 目标：项目能跑起来，在浏览器能看到页面

#### 任务清单
- [ ] 创建完整的项目目录结构

- [ ] 创建 config.py（含DeepSeek API配置位）

## Phase 1：用户系统（预计2天）
### 目标：用户可以注册、验证邮箱、登录、退出

#### 任务清单
- [ ] 创建用户数据库表（users）
- [ ] 实现注册页面（表单：用户名、邮箱、密码）
- [ ] 实现邮箱验证（发送验证邮件+验证链接）
- [ ] 实现登录/退出功能
- [ ] 实现个人主页（显示基本信息）
- [ ] 注册时提醒"请检查垃圾邮件"
- [ ] 验证：可以注册→收验证邮件→登录→看到个人信息

#### 配置要求
- 在 config.py 中配置邮箱信息（发件邮箱、SMTP服务器）
- 在 config.py 中配置 DeepSeek API Key 位置

## Phase 2：书搜搜核心（预计3天）
### 目标：用户输入prompt，AI推荐图书并显示状态

#### Step 2.1 图书馆模拟API（0.5天）
- [ ] 在 modules/search/library_api.py 中模拟图书馆数据
- [ ] 模拟数据包含10+本图书，覆盖三种状态
- [ ] 实现搜索接口函数（根据关键词返回图书列表）
- [ ] 验证：调用函数能返回正确的格式

#### Step 2.2 AI推荐功能（1天）
- [ ] 在 config.py 中填写DeepSeek API配置
- [ ] 实现AI调用函数（发送prompt → 获取推荐书单）
- [ ] 实现：从推荐书单提取关键词 → 查图书馆API
- [ ] 实现：合并两个结果

#### Step 2.3 搜索结果页面（1天）
- [ ] 卡片样式展示搜索结果
- [ ] 三种状态分别显示（绿色/黄色/红色标签）
- [ ] 评论接在书籍信息后面
- [ ] 中英双语切换
- [ ] 验证：输入prompt → 看到带状态的卡片

#### Step 2.4 论坛数据联动（0.5天）
- [ ] 搜索时同时查论坛有没有相关评论
- [ ] 在卡片下方显示评论摘要
