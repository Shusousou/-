## 1. 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 后端框架 | FastAPI | Python异步Web框架，性能好、自动生成API文档 |

| 数据库 | SQLite | 开发阶段使用，后续可迁移至MySQL |
| AI模型 | DeepSeek API | 用户提供API Key |

| 运行环境 | Python 3.10+ | |

## 4. 关键设计决策

### 4.1 图书馆API封装
- 所有图书馆API调用封装在 library_api.py
- 后续对接真实API，只需改这一个文件
- 其他模块不需要知道API实现细节

### 4.2 模块间数据流通
- 通过函数调用互通数据（简单直接）
- 不使用消息队列等复杂方案
- 所有模块通过 ISBN 字段关联图书数据

### 4.3 数据库
- 开发阶段使用SQLite（零配置，适合新手）
- 通过 SQLAlchemy ORM 操作数据库（后续迁移方便）

## 5. AI调用方案

### 5.1 DeepSeek API 调用
`python
# 请求格式
POST https://api.deepseek.com/v1/chat/completions
Headers:
  Authorization: Bearer sk-xxx
  Content-Type: application/json

Body:
{
  "model": "deepseek-chat",
  "messages": [
    {"role": "system", "content": "你是一个图书推荐助手..."},
    {"role": "user", "content": "用户输入的prompt"}
  ]
}
`

### 5.2 AI的三个用途
1. **书搜搜**：分析用户输入 → 生成推荐书单 + 关键词
2. **交换借阅**：协助用户写借出要求和权益提示
3. **Request匹配**：提取关键词 + 扩展搜索

## 6. 邮件系统
- 使用 smtplib（Python内置）
- 邮箱验证：发送验证链接
- 通知邮件：Request匹配成功时通知用户
