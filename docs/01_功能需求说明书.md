## 1. AI图书推荐（核心功能）

### 1.1 整体流程
`
用户输入Prompt → AI分析 → 生成推荐书单+关键词
  → 关键词搜索图书馆API → 获取馆藏状态
  → 同时查询论坛（有没有人评论过？）
  → 同时查询交换（有没有人愿意借？）
  → 展示完整结果（卡片样式，评论接在书籍信息后面）
`

### 1.2 AI推荐生成
- 用户输入自然语言（如："我想看关于人工智能的书"）
- AI分析后生成：
  - **推荐书单**：书名 + 作者 + ISBN + 推荐理由
  - **搜索关键词**：用于查询图书馆API

### 1.3 三种结果处理
| 情况 | 显示内容 | 样式 |
|------|---------|------|
| ✅ 图书馆有且在馆 | 位置信息 + 推荐理由 | 绿色标签 |
| ⏳ 图书馆有但借出 | 预计归还日期 + 预约方法 | 黄色标签 |
| 📝 图书馆无匹配 | 引导用户向图书馆反馈需求 | 红色标签 |

### 1.4 AI搜索策略
1. 先用关键词精确搜索（快）
2. 结果太少时 → AI扩展搜索（同义词/近义词，防止漏搜）

### 1.5 数据联动
- 搜索结果中显示论坛相关评论（评论接在书籍信息后面）
- 搜索结果中显示是否有书友愿意交换借阅

## 2. 书籍交换借阅

### 2.1 放书人流程
`
用户登录（填写基本信息，包含邮箱）
  → 点击"发布图书"
    → 填写：书名、借阅要求、期望
    → AI辅助：帮写要求、提醒权益保护
  → 发布成功，显示在可借列表中
`

### 2.2 借书人流程
`
浏览可借书籍列表 / 搜索框搜索
  → 点击"我想借这本书"
    → 进入站内临时评论区沟通
    → 双方交换邮箱，私下联系
`

### 2.3 Request匹配
`
用户挂Request（如："有关科技的书"）
  → AI提取关键词
  → 每上新书 → 检查是否匹配关键词
  → 匹配成功 → 发送邮件通知用户
`

### 2.4 搜索联动
- 在书搜搜搜索时，如果图书馆没有，但有人愿意交换，显示交换信息
- 在交换模块搜索时，可以使用书搜搜的搜索功能

## 3. 好书推荐论坛

### 3.1 功能列表
| 功能 | 说明 |
|------|------|
| 发帖 | 填写书名 + ISBN + 分类 + 评论内容 |
| 回帖/讨论 | 在帖子下方回复讨论 |
| 点赞 | 对喜欢的帖子点赞 |
| 分类浏览 | 按书籍分类查看帖子 |
| 搜索 | 按书名/作者/关键词搜索帖子 |
| 关联ISBN | 帖子关联书籍ISBN，便于数据互通 |

### 3.2 与书搜搜的联动
- 书搜搜搜索结果中，自动显示论坛相关评论
- 论坛帖子展示时，自动查询图书馆API显示该书的在馆状态

## 4. 非功能需求

### 4.1 用户系统
- 注册：用户名 + 邮箱 + 密码
- 邮箱验证（提醒检查垃圾邮件）
- 登录/退出
- 密码加密存储

### 4.2 界面
- 中英双语切换（默认中文）
- 卡片样式展示结果
- 响应式布局（电脑/手机都能看）

### 4.3 数据
- 开发阶段使用SQLite
- 图书馆API先用模拟数据，后续对接真实API
- 所有模块通过ISBN字段互联
