# 书搜搜 — 开发日志使用说明

## 为什么写日志？
- 记录每天做了什么（方便回溯）
- 记录遇到的问题（方便排查）
- 记录明天的计划（不遗漏任务）
- 记录决策原因（为什么这样做）

## 怎么写日志？

### 格式
```
# YYYY-MM-DD 开发日志

## 今日完成
- [x] 完成了xxx功能
- [x] 修复了xxx bug

## 遇到的问题
- 问题：xxx  →  解决：xxx

## 明日计划
- [ ] 做xxx
- [ ] 做xxx

## 备注/想法
- 想到了xxx可以改进
```

### 规则
1. 每天一条日志，文件名格式 YYYY-MM-DD.md
2. 先复制模板，再填写内容
3. 完成就打勾，没完成移到第二天
4. 遇到问题一定要记，哪怕没解决

### 日志文件位置
所有日志文件保存在项目根目录的 logs/ 文件夹中。

### 查阅方式
直接用文本编辑器打开 logs/ 目录下的文件即可。
